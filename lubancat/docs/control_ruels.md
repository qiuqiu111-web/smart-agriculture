# 奶油生菜智能灌溉系统 - 设计与编码约束白皮书

> **目标定位**：摒弃传统的"土壤阈值单一触发"逻辑，建立以 **VPD（饱和水汽压差）** 为核心驱动、**时段策略** 为骨架的精准灌溉模型。本系统不仅用于控制1-2株生菜的生长，更是为了采集高质量的"决策过程数据"，为后续Linux端AI模型训练提供可靠的专家规则基准。

---

## 一、 核心思想

1. **水分角色的转变**：将水从"生存必须品（干了就浇）"转变为"生长调节剂（根据蒸腾需求精准分配）"。
2. **驱动源切换**：决策的核心驱动力从 `soilHumidity`（滞后指标）转移到 **`VPD`（实时蒸腾拉力指标）**。土壤湿度退化为"安全底线"与"水库缓冲池"。
3. **时段优先于阶段**：项目周期较短，弱化"生长阶段（幼苗/旺长/采收）"的多周跨度差异，聚焦于**单日内的时段节律**——植物在一天中不同时间的生理状态差异远大于跨周的土壤目标差异。

---

## 二、 当前物理条件与硬性约束

1. **极小空间与单株效应**：面积仅 110cm * 80cm，植株仅 1-2 株。系统的"水惯性"极小，任何微小的水量波动都会被传感器迅速捕捉，极易引起控制震荡。
2. **致命的"软管直冲"陷阱**：基础抽水电机流量极大（预估 >100L/H），若直冲土壤，1秒钟的水量足以造成局部径流或冲刷，导致传感器数据失真。
   - **强制约束**：代码中必须将水泵的执行单位定义为**"极短时间的点动（初步定为 0.5 秒）"**，且物理端必须改造出水口（如针孔滴灌或缓冲瓶盖），将水流形态变为"水滴"而非"水柱"。
3. **传感器的空间错位风险**：电容式传感器探测范围有限。必须确保出水滴点落在传感器的探测半径内，否则代码记录的"浇水后湿度变化"将是负逻辑。
4. **实验室光照不确定性**：
   - 白天靠窗自然光，受云层遮挡影响剧烈，光照数据具有高波动性。
   - 晚上可能开室内灯工作，光照读数会出现"伪白天"信号。
   - **结论**：光照不再作为独立的一票否决条件，而是与时间联合推断"有效光合窗口"。
5. **控制周期基准**：考虑到水分在微小土壤体积中的扩散时间（约5-10分钟），以及防止继电器频繁烧毁，**系统的决策采样周期强制设定为 10 分钟一次**。

## 三、 数据预处理与特征工程（输入层）

STM32采集到原始数据后，在进入规则引擎前，必须先进行语义转化：

```
原始输入: {temperature, humidity, soilHumidity, illuminance, time}

转化特征:
  VPD (饱和水汽压差) = f(temperature, humidity)
      → 核心决策变量。衡量大气从叶片"抽取"水分的力度，单位 kPa。
      → 公式采用 Murray (1967)：Es = 0.61078 * exp(17.27 * T / (T + 237.3))
          VPD = Es * (1 - RH/100)

  Time_Window (时段分类)
      → 根据 hours + minutes 判定当前生理时段。
      → 这是策略选择的"骨架"，决定当前应该采取什么姿态。

  Light_Flag (光合活性推断)
      → 光照不再作为定量驱动力，而是与时间联合做二元推断：
        如果 (Time_Window == 白天段) AND (illuminance > 低阈值) → 光合活跃
        如果 (Time_Window == 夜间段) OR (illuminance < 极低值)  → 光合停止
      → 夜晚开灯的"伪白天"由 Time_Window 兜底锁死，不会被光照骗开浇水。

  Soil_Buffer (土壤缓冲状态)
      → 当前土壤湿度距离目标值的差值。
      → 目标值简化为单一区间（不区分生长阶段），见第四节。
```

## 四、 每日时段策略表

> 这是整个控制系统的核心决策骨架。一天被划分为 6 个时段，每个时段对应不同的生理状态和浇水策略。

| 时段 | 时间范围 | 生理状态 | 水分需求 | 浇水策略 |
|------|---------|---------|---------|---------|
| **清晨启动期** | 06:00 - 08:00 | 光照渐起，气孔开始张开，蒸腾启动。植物从夜间"休眠"转入"日间光合模式"。 | 中等。如果前一日傍晚未补水且土壤偏低，此时是补水的最佳窗口。 | **预防性补水窗口**：若 `Soil_Buffer` 低于目标区间下限，执行 1 次脉冲补水，为即将到来的高蒸腾时段储备水分。 |
| **上午光合期** | 08:00 - 12:00 | 光合速率快速上升，蒸腾拉力增强，VPD 进入理想区间。植物水分利用效率最高。 | 高。这是植物真正"喝水"的时段，水分被有效用于光合作用。 | **VPD 主动驱动**：若 `VPD > 解锁阈值` 且 `Soil_Buffer` 允许，执行脉冲补水。这是最有价值的浇水时段——水被用于生长而非蒸发。 |
| **正午保护期** | 12:00 - 14:00 | 温度达峰，若温度过高（>30°C）气孔部分关闭以自我保护，蒸腾反而下降。地温过高时浇水会导致根系窒息（炸根）。 | 若温度正常则中等；若高温则植物主动"断水"。 | **条件锁死**：若 `temperature > 30°C`，强制跳过，不浇水。若温度正常且 VPD 驱动，可少量补水。 |
| **下午光合期** | 14:00 - 17:00 | 光照/温度从峰值回落，蒸腾仍持续但强度减弱。植物恢复水分吸收。 | 中等。午后是光合作用的"下半场"，但效率低于上午。 | **VPD 主动驱动（降级）**：VPD 阈值比上午更高（更保守），仅在土壤明显偏低时响应。 |
| **傍晚回补期** | 17:00 - 19:00 | 光照快速衰减，温度下降，蒸腾减弱。植物转入"储备模式"——将水分储存在组织中供夜间使用。 | 中等偏低。不需要大量水，但需要补充白天消耗的土壤水分储备。 | **黄金回补窗口**：若 `Soil_Buffer` 低于目标中线，执行 1 次脉冲补水。这是度过无光夜晚前最后一次安全补水时机。 |
| **夜间休眠期** | 19:00 - 06:00 | 无光合作用，气孔接近关闭，蒸腾极低。叶片表面若有自由水 + 无风 = 100% 诱发病害（霜霉病/软腐病）。 | 极低（仅维持细胞渗透压）。 | **硬锁死**：除 `Emergency_Save` 外，绝对不浇水。VPD、土壤湿度均不决策。 |

### 策略总结

```
┌──────────────────────────────────────────────────────────────┐
│  时段骨架决定"能不能浇"（权限）                                │
│  VPD 决定"该不该浇"（需求强度）                                │
│  Soil_Buffer 决定"还有多少余地"（安全余量）                    │
└──────────────────────────────────────────────────────────────┘
```

| 决策变量 | 角色 | 权重 |
|----------|------|------|
| Time_Window | 权限控制（一票否决/一票放行） | 最高 |
| VPD | 浇水需求强度（越高越需要浇） | 核心 |
| Soil_Buffer | 安全底线 + 防过量 | 约束 |

## 五、 VPD 驱动的浇水决策（非边界条件下的核心逻辑）

### 5.1 VPD 计算公式

采用 **Murray (1967)** 公式，精度满足农业需求，计算量小：

```
Es = 0.61078 × exp( 17.27 × T / (T + 237.3) )    // 饱和水汽压 (kPa)
VPD = Es × (1 - RH / 100)                          // 饱和水汽压差 (kPa)
```

其中 `T` = 空气温度 (°C)，`RH` = 空气相对湿度 (%)。

### 5.2 VPD 区间与生菜响应

| VPD 范围 (kPa) | 含义 | 浇水姿态 |
|----------------|------|---------|
| < 0.3 | 空气接近饱和，蒸腾几近停止，浇水只会导致沤根 | **不浇** |
| 0.3 - 0.8 | 舒适区间，蒸腾温和。适合幼苗/湿度敏感期 | 仅在土壤偏低时微量补水 |
| 0.8 - 1.5 | 理想蒸腾区间，水分利用效率最高 | **主动浇水**（前提是时段允许且土壤不涝） |
| 1.5 - 2.0 | 蒸腾胁迫出现，植物开始加大抽水力度 | 若时段允许且土壤低于安全线 → 补水 |
| > 2.0 | 严重干旱胁迫，叶片可能开始失水 | 无条件补水（标记为高优先级） |

### 5.3 VPD + 时段联合决策矩阵

```
当 Time_Window 允许浇水时，按以下优先级判定：

  if VPD > 2.0 kPa  → 高优先级补水（胁迫响应）
  elif VPD > 0.8 kPa AND Soil_Buffer < 目标中线 → 正常补水（VPD驱动）
  elif VPD > 0.3 kPa AND Time_Window == 傍晚回补期 → 预防补水（储备过夜）
  elif VPD < 0.3 kPa → 不浇（空气窒息锁）
```

## 六、 边界条件（硬锁死，一票否决权）

以下条件只要满足其一，直接跳过本次决策，`Pump_Action = 0`，并记录 `rule_triggered`。

| 编号 | 锁名称 | 条件 | 原因 |
|------|--------|------|------|
| L1 | **夜间休眠锁** | `Time_Window == 夜间休眠期 (19:00-06:00)` | 无光合、无蒸腾、无风，浇水必然诱发病害 |
| L2 | **正午高温锁** | `Time_Window == 正午保护期 (12:00-14:00)` AND `temperature > 30°C` | 地温过高，突然加水导致根系窒息（炸根） |
| L3 | **空气窒息锁** | `VPD < 0.3 kPa`（等价于 RH 极高） | 空气中无水分梯度，植物不蒸腾，浇水 = 沤根 |
| L4 | **防重入锁** | 当前 10 分钟周期内已执行过浇水 | 防止继电器频繁烧毁 + 等待土壤水分扩散 |
| L5 | **救命底线锁（唯一越权条件）** | `soilHumidity < 15%`（数值待标定） | **无视 L1-L4 全部限制**，强制执行 0.5 秒补水，日志标记 `Emergency_Save` |

---

## 七、 输出执行约束

- **摒弃连续量计算**：不要试图用公式算出"缺了20毫升水，水泵需开3.14秒"。在小空间、软管滴灌的非线性系统中，这种计算毫无意义且极度危险。
- **唯一输出格式**：`Pump_Action` 只有两种状态：`0` (关闭) 或 `1` (开启 0.5 秒脉冲，执行后自动归零)。
- **物理防冲刷保护**：在代码逻辑中确保单次触发不能超过 0.5 秒（需等水泵实际流量标定后确认）。
- **最小浇水间隔**：两次 `Pump_Action = 1` 之间至少间隔 2 个决策周期（20 分钟），给土壤充足的水分扩散时间。

## 八、 AI 数据采集标注规范（Linux端落库要求）

为了确保未来 AI 能够学到"超前控制"和"状态机逻辑"，而不是学到一个粗暴的线性映射，数据库每条记录必须包含以下字段：

### 8.1 传感器原始值（来自 STM32）
| 字段 | 类型 | 说明 |
|------|------|------|
| `timestamp_ms` | INT64 | Linux 端接收时间戳（毫秒） |
| `temperature` | FLOAT | 空气温度 (°C) |
| `humidity` | FLOAT | 空气相对湿度 (%) |
| `soil_humidity` | FLOAT | 土壤湿度（电容传感器原始值） |
| `illuminance` | FLOAT | 光照强度 (lux) |

### 8.2 特征工程值（Linux 端计算）
| 字段 | 类型 | 说明 |
|------|------|------|
| `calculated_vpd` | FLOAT | Murray 公式计算的 VPD (kPa) |
| `time_window` | STRING | 当前时段分类（见第四节表格第1列） |
| `light_flag` | BOOLEAN | 光合活性推断（时间+光照联合判定） |
| `soil_buffer` | FLOAT | 当前湿度与目标中线的差值 |

### 8.3 决策过程记录
| 字段 | 类型 | 说明 |
|------|------|------|
| `rule_triggered` | STRING | 本次周期命中的规则（如 `Night_Lock`, `Noon_Lock`, `VPD_Drive`, `Evening_Replenish`, `Emergency_Save`） |
| `action_type` | STRING | `Null` / `VPD_Active`（VPD驱动）/ `Preventive`（预防回补）/ `Emergency_Save`（救命） |
| `pump_action` | INT | 0 或 1（脉冲） |

### 8.4 数据质量标记
| 字段 | 类型 | 说明 |
|------|------|------|
| `data_reliability` | BOOLEAN | 浇水后短时内土壤湿度未扩散均匀，标记为 `false`，防止 AI 学到"浇水导致湿度下降"的幻觉 |

## 九、 土壤目标值（简化为单一区间）

项目周期较短，暂不区分生长阶段，采用统一的目标区间：

| 参数 | 值 | 说明 |
|------|-----|------|
| 土壤目标中线 | 45% | 日常维持目标 |
| 允许下限 | 35% | 低于此值且时段允许 → 触发补水 |
| 允许上限 | 55% | 高于此值 → 即使 VPD 驱动也不浇水（防涝） |
| 救命底线 | 15% | 低于此值 → 无视一切锁死，强制执行 Emergency_Save |

> **延伸规划**：若未来需要区分生长阶段，可将目标值改为可配置的三段式（幼苗 35%±5 / 旺长 50%±5 / 采收 40% 上限 45%），通过 config 注入。
